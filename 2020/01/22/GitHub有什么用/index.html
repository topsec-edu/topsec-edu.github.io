GitHub有什么用

GitHub使用方法

为什么要学习GitHub

GitHub 对于技术开发人员来说很有帮助，上面有很多优秀的开源项目。

在实际开发中，像git和svn这些代码管理工具是必然会用到的。

不管是应用在实际项目上，还是对源码的学习，这都是提升我们自己技能和开拓眼界的一个重要渠道。

我们在GitHub上使用最多的除了开源自己的项目之外，就是利用GitHub找到自己想要的开源项目学习了。

GitHub基本使用

首先创建一个项目，我就不说怎么创建项目了。百度

简单介绍github的使用

Code是项目代码，在这一栏里可以看到项目描述，下载项目源码

issues可以理解为评论区，但并不仅仅是这个功能。更是一种bug跟踪器，为你的项目跟踪任务(tasks)、强化(enhancements)以及bug的利器。issues也可以反应一个开源项目的活跃度。

Pull requests是你自己修改源码后，可以发起请求让仓库主人决定是否通过

Projects可以永来组织和排列项目开发的优先级

Wiki可以用来当做自己的知识宝库

insight可以看到项目的一些数据，贡献者的数量，访问数，clone数等等。

setting可以设置仓库名，GitHub Pages，删除仓库等。

如何搜索开源项目

Explore（探索）

点击Explore GitHub



GitHub 会把所有近期有活跃的项目呈现给大家，是没有经过筛选的，按照默认排序。这时候，点击右上角的 Trending（趋势） 按钮，根据你的需要筛选语言、更新日期等。

Search

除了 Trending ，还有一种最主动的获取开源项目的方式，那就是 GitHub 的 Search 功能。

举个例子，你是做 iOS 的，接触 GitHub 没多久，那么第一件事就应该输入 iOS 关键字进行搜索，然后右上角选择按照你想要的结果来排序，结果如下图：

1、根据仓库的标题

in:name java

2、根据仓库的描述

in:description javascript

3、根据README查找关键词

in:readme javascript

4、根据start、fork数量大小查询

stars:>3000 spring cloud stars数大于3000的

fork:>1000 spring cloud fork数大于3000的

5、根据仓库更新时间

pushed:>2019-01-01 spring cloud 更新时间在2019年1月1日之后的

6、根据仓库LICENSE搜索 开源证明

license:apache-2.0

7、根据某人或组织或编程语言搜索

user:GitSuperDrew languages:java

org:hibernate

找到开源项目后

打开后，项目里会有一个标准的说明文件，来说明项目的相关情况，比如如何使用，需要什么其他库文件，项目的相关说明等等。基本上每个项目都会有这个标准的说明

也可以点击右上角的下载按钮，把代码下载回本地电脑再慢慢研究。

也可以点击上方的Fork按钮，创建一个属于自己的分支，可以在自己的分支上修改代码，而不会影响原来的。

总结

把刷微博、朋友圈的时间，静下来刷GitHub，总有一天你会发现，自己也可以成为大牛！
